# Contribuyendo a [Nombre de tu Proyecto]

¡Gracias por tu interés en contribuir! Este proyecto es open source y abierto a colaboraciones. A continuación, te explicamos cómo puedes ayudar de forma efectiva.

## Cómo Contribuir

Puedes contribuir de varias formas: reportando bugs, proponiendo features, mejorando la documentación o enviando código.

### Reportar Bugs

- Usa la sección de [Issues](https://github.com/Maya-AQSS/maya-signer/issues) para reportar problemas.
- Incluye: descripción clara, pasos para reproducir, versión del software y capturas de pantalla si aplica.
- Etiqueta como "bug".

### Proponer Features o Mejoras

- Abre un Issue con etiqueta "enhancement" o "feature".
- Describe el problema que resuelve y por qué es útil.
- Si es posible, incluye un borrador de implementación.

### Enviar Código (Pull Requests)

1. Haz un fork del repo y clona tu fork localmente.
2. Crea una branch: `git checkout -b feature/nueva-funcion`.
3. Implementa tus cambios. Sigue el estilo de código (ej: PEP8 para Python).
4. Añade tests si aplica.
5. Commit con mensajes claros: `git commit -m "Añade nueva función X"`.
6. Push a tu fork: `git push origin feature/nueva-funcion`.
7. Abre un Pull Request (PR) desde tu branch a `main` del repo original.
   - Describe los cambios, referencia Issues relacionados.
   - Espera review; puede requerir ajustes.

Tu PR será revisado lo antes posible. ¡Gracias por tu contribución!

## Guía de Estilo

- Usa [PEP8](https://pep8.org/) para Python (o el estilo del proyecto).
- Añade comentarios y docstrings.
- Mantén el código limpio y legible.
- Siempre que puedas utiliza anotaciones.
